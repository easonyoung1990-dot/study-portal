# CLAUDE.md

本文件为 AI 助手（Claude Code 等）提供本仓库的工作指南。开始任何改动前请通读一遍。
完整产品背景见 `docs/产品蓝图.md`，本文件是它的工程化速查版。

## 这是什么

**家庭学习门户（study-portal）**：把散落的学习资产收拢成一个"自家学习 App"，
全家任何设备打开网址即可使用。两个孩子各自独立：**Kenton（🦁 青蓝）** 与 **Eddey（🐯 珊瑚橙）**。

核心架构是**静态只读**：

```
展示层  GitHub Pages 静态站（孩子端只读打开）
数据层  仓库内 data/*.json（任务/软件/积分/播报索引）
生产层  爸爸电脑上的 Claude Code（改 JSON → git push → 约 1 分钟后网站自动更新）
```

> 关键认知：**孩子端永远只是打开看，所有数据更新都发生在爸爸电脑端。**
> 网站没有后端、没有账号系统、没有孩子端写入——不要尝试引入这些。

## 隐私红线（强制，最高优先级）

仓库是**公开**的（免费 GitHub Pages 要求），因此全网可见。任何改动都必须满足：

1. **只用英文小名 Kenton / Eddey**。绝不出现真实姓名、姓氏、学校、班级、地址、照片。
2. **敏感明细只存本地** `local-data/`（已被 `.gitignore` 屏蔽，永不上传）：
   错题原文、批改记录、成绩单等。网站上只放**匿名化统计**（如"小数除法正确率 62%→85%"）。
3. **永不提交任何账号、密码、密钥、token**。家长页也不放任何凭据。
4. 提交前自检：新增/修改内容是否泄露了上述任一项？有疑问就停下来问，不要先推送。

## 技术栈与约束

- **纯静态**：原生 HTML + CSS + 原生 JS（`fetch` + 模板字符串）。**无框架、无构建、无依赖、无 `package.json`、无 `node_modules`。**
- 不要引入打包器、npm 包、CSS 框架或 JS 框架。保持零依赖、可直接用浏览器打开。
- 所有页面共用 `assets/styles.css`（设计基线：**窄屏单列优先**、大按钮、糖果卡通风、孩子零打字）。
- 语言：界面与文档为中文；学习软件（`apps/`）内容可英文。

## 配音（朗读语音）：一律用神经网络真人音

**今后所有带朗读/报读的软件**（默写、听写、英语阅读、古诗、播报等，中英文都算）——**默认用预合成的神经真人音，不要用浏览器自带 TTS 当主音**（浏览器 TTS 机器味重、各浏览器发音不一致，仅作缺失时的自动回退）。

- **怎么做**：构建时（爸爸电脑这个"生产层"）用 edge-tts 把该软件要朗读的词句预合成成 MP3，按内容哈希命名放到 `apps/audio/<软件名>/`；网页里的朗读函数（如 `speakOnce`）先播对应 MP3、缺失才回退浏览器 TTS。**页面本身仍是零依赖纯静态**——MP3 只是静态资源，和 `data/*.json` 一样在生产层生成后随仓库发布，不违反"无构建/无依赖"。
- **音色**：英文 `en-US-AriaNeural`、中文 `zh-CN-XiaoxiaoNeural`；默写/听写类英文（必要时中文）放慢约 `-10%` 方便孩子书写。换音色按需（中文男声 `zh-CN-YunxiNeural`、英音 `en-GB-SoniaNeural` 等）。
- **工具**：本机 Python `C:\Users\User\AppData\Local\Programs\Python\Python313-arm64\python.exe`（已装 edge-tts）；生成脚本 `C:\Users\User\broadcast\gen_audio.py`（从 HTML 自动提取要读的文本）或 `gen_audio_manifest.py`（按每个软件的 `apps/audio/<名>/_manifest.json` 批量合成）。
- **关键一致性**：网页 JS 的 `audioHash()` 与脚本 `fnv()` 必须产出完全一致的哈希（对 `lang+"|"+text` 的 UTF-8 字节做 FNV-1a，输出 8 位十六进制）；改任一处都要同步另一处，否则音频全部 404、退回机器音。
- **范本**：照抄 `apps/eddey_english.html` 或 `apps/news_20260615.html` 里已跑通的写法（`AUDIO_DIR` + 可复用音频元素 + `audioTok` 打断保护 + audio-first 回退 + 默写"读 N 遍"）。

## 页面如何读取数据

每个页面用 `fetch(..., { cache: "no-store" })` 读取对应 JSON 并渲染。约定：

- 孩子标识统一用 `who` 查询参数：`kenton` / `eddey`（如 `today.html?who=kenton`）。
- 两个孩子的主题色取自 CSS 变量 `--kenton` / `--eddey`，**内容与配色绝不混用**。
- 学科颜色/图标映射在 `apps.html` 的 `SUBJECT_META` 中（英语/语文/古诗/数学）。

## 数据文件格式（改动时严格遵守现有字段）

```jsonc
// data/tasks.json —— 今日任务（爸爸端每晚生成次日版）
{ "date": "2026-06-12",
  "kenton": [ { "type": "app", "title": "...", "link": "apps/xxx.html", "stars": 3 },
              { "type": "pdf", "title": "...", "link": "papers/xxx.pdf", "stars": 5 } ],
  "eddey":  [ ... ] }              // type 取 "app"(📖) 或 "pdf"(📄)

// data/apps.json —— 软件库索引（新软件入库时追加一条）
[ { "child": "kenton", "subject": "英语", "title": "...",
    "file": "apps/xxx.html", "added": "2026-06-11" } ]

// data/scores.json —— 积分（唯一可写来源：爸爸端汇总）
{ "updated": "2026-06-11", "kenton": { "week": 18, "total": 236 },
                           "eddey":  { "week": 21, "total": 214 } }

// data/podcast.json —— 播报索引（每期一条；today.html 取数组最后一条为"最新"）
[ { "date": "2026-06-11", "file": "podcast/2026-06-11.html", "topics": ["NBA", "太空"] } ]
```

约定：`child`/`who` 仅限 `kenton` 或 `eddey`；日期用 `YYYY-MM-DD`；
`link`/`file` 为相对仓库根的相对路径；JSON 必须合法（改完务必校验）。

## 学习软件标准模板（做新软件必须内置 ⭐）

`apps/` 下每个**练习/做题类**软件，都必须内置以下三件套（零依赖、纯前端实现），
不要再省略。参考实现见 `apps/eddey_chinese_review.html`、`apps/kenton_chinese_review.html`：

1. **声音**：答对/答错音效（Web Audio `AudioContext` 即时合成，无音频文件）、
   `🔊读题`（`speechSynthesis` 中文 TTS）、`🔔/🔕` 静音开关。
   注意 `AudioContext` 需用户手势后 `resume()`。
2. **计时**：答题区实时计时 `⏱`（`setInterval` 读 `Date.now()-t0`），
   结果页显示总用时与平均每题用时；每题用时（ms）记进数据。
3. **记录面板 + 导出**：用 `localStorage` 累积每题 `{word(答案),hint(题干),ok,ms,user(所选/自评),tag(错因)}`；
   首页放"我的练习记录"面板（显示已练关数/题数/错题数）+「复制」「存成文件」按钮。
   **导出格式必须兼容**：文本含"反复错的题/每关明细"，并附
   `{"child":"<哥哥|弟弟>","subject":"...","events":[...]}` 原始 JSON——
   这样导出后投喂回来，能用同一套流程归档进 `data/learning/` 并刷新学情画像。
4. **少打字 / 可口述（⭐ 两娃打字还不熟）**：题型优先用**点选**；确需作答的填空题要给
   **🎤 语音输入**（浏览器自带 `SpeechRecognition`/`webkitSpeechRecognition`，纯前端零依赖）：
   点麦克风说出答案 → 自动转文字填入；识别结果与正确答案规范化比对，**听对了自动判分**，
   听岔了可再说一次或打字兜底。不支持语音的设备自动隐藏麦克风、回退到打字/点选。
   英语用 `lang="en-US"`、语文用 `lang="zh-CN"`。参考实现见 `apps/eddey_english_review.html`。
5. **错题必有解释（⭐ 必备，不可省，最高优先级之一）**：每道客观题（选择/判断/填空等）
   都必须带 `explain`——讲清"**为什么是这个答案 / 为什么错**"（如多音字列出各读音、形近字辨偏旁、
   关联词点明关系类型、课文主旨给出依据）。答完（尤其答错）必须把解释显示给孩子看；
   结果页的错题回顾也要带上解释。主观题（默写/仿写/写作）给"参考答案 + 评分要点(checklist)"。
   **绝不允许只判对错、只给正确答案而不讲原因。** 出题或做新软件时，`explain` 和题目本身同等必填。
6. **每关显示对错数（⭐ 必备）**：闯关/选关地图上，每一关/每个模块练过后都要显示**上次成绩 ✅对X ❌错Y**
   （读 `localStorage` 历史事件，按 lesson 名取最近一次）。让孩子和家长一眼看到哪关错得多。
   参考：`apps/eddey_chinese_review.html`、`apps/kenton_chinese_review.html` 的 `lessonDone()` + done 角标。
7. **生词默写/听写 = 用"默写助手"那套机制（⭐ 凡是默写类软件都按它做）**：参考实现
   `apps/kenton_dictation.html`（哥哥实测好用的版本，务必沿用）。要点：
   - **全自动报默**：TTS 把每个词/句**连读 3 遍**（语文读词语、英语读英文，**不显示中文、不偷看**），
     配**大倒计时 + 进度条**，时间到自动跳下一个；按词长自动估时（`wordSec`）；带 `🔁再听一遍/⏸暂停`。
   - **答案对照（判分必须可靠 ⭐）**：一课默完一次性出示全部答案，孩子**把写错的点一下标红**（其余算写对）、可 `🔁只重默写错的`。
     **关键＝错词永久记录**：凡标红过一次的词必须进"错词本"持久保存（见错词重做），不能因为某次没标红就丢失——
     真正杜绝"假全对污染学情"的是这条持久化，而不是强迫逐词点击。
   - **理解小测**（可选但建议）：语文选词填空、英语中英互译，检查"真懂"而非死记。
   - **归档**：事件 `{ts,type:'dict'|'quiz',subject,lesson,total,wrong,durationSec,items:[{word,hint,ok,ms}]}`，
     聚合出"反复错的词(错≥2)"，一键复制/下载（即上文导出格式）。
   - **错词重做 + 错词本（⭐⭐ 核心要求，所有默写/听写软件必须内置）**：每个默写软件都要带一个**持久错词本**（`localStorage` 独立键，如 `mx_wrongbook_<app>`）：
     机制＝**默错过一次就进错词本、永不自动丢失**；错词**连续答对 2 次（第一次答对不算）才算掌握、自动移出**；错词本可自动吸收主默写软件归档里答错的词。
     实现：**包裹 `logEvent`**（每次默完都经过它）来记录——`dict` 事件里 `ok=false` 的词 `wbAddWrong`、`ok=true` 的词 `wbAddRight`（连胜+1，≥2 标记掌握）；**学习档案页与导出文本都要列出错词本**（不只是"反复错≥2"），并提供 `🔁重默错词`(`startWrongbook()`)。统一注入脚本见 `tools/add_wrongbook_to_dictation.py`。**主默写界面外观不变。**

> 这套模板让"孩子端本地记录 → 导出 → 爸爸投喂 → 进化"的闭环对每个软件都成立。
> 架构是静态只读，**没有也不要加后端自动回传**；同步靠导出这一步人工搬运。

## 出题难度与格式标准（强制 ⭐，2026-06-19 定）

**今后所有卷子/复习软件/练习内容，难度一律对标上海《一课一练·增强版》，且"讲解与题目合在一起"（讲练合一）。** 这是家长定的硬标准，不达标即返工。

### 难度对标"一课一练·增强版"
- **难度梯度**：每个板块都要有 **基础 → 巩固 → 拔高（标 ★）** 三档；每板块至少 1–2 道拔高/拓展题。不能只停在"认一认、背一背、套规则"的基础题。
- **题型进阶**（按学科取用）：
  - 语文：**理解性/情境默写**（按提示语境填句，非纯背）、转述句**进阶**（含时间地点词 今天→那天/明天→第二天/这→那、反问↔陈述、引述含问句/感叹/命令）、词语**辨析**（近义辨析/语境选词/多义字/成语语素）、病句、标点、修辞**赏析**、**课内+课外**阅读（梯度设问：词句理解→概括→赏析→开放联系生活）、语言运用/综合实践、小练笔。
  - 英语：时态/句式**综合**转换、对划线提问全类型、阅读细节+推断、情境写作、易错点对比辨析。
  - 数学：多步应用题建模、一题多解/巧算、几何拆补、逆向反推、易错对比题、拓展拔高★。
- **覆盖**：紧扣该孩子 `data/learning/` 错题与教材单元；把真实错题融进来，但要在其上**加难一档**，不能只是把原错题抄一遍。

### "讲练合一"格式（讲解和题目必须在一起）
每个板块按 **【方法点拨/知识清单】（口诀/规则/易错提醒）→【例】带解析的例题 →【练】梯度练习** 组织，讲解就排在题目旁边/上方，**不要把讲解和题目分家**。答案区给"**答案+简要解析**"（讲清为什么），不能只给答案。软件里则用"关卡 intro 方法讲解 + 每题 explain 解析 + 错题回炉"实现讲练合一。

### 仍须遵守
儿童友好、窄屏可打印、隐私红线（只用小名、错题原文不入公开仓库）、零依赖纯静态；卷子产出 A4 可打印 PDF（生成器放 `tools/`，可复现）。

## 学习软件设计哲学（强制 ⭐，2026-06-19 定）

**软件绝不能只是"刷题机器"。** 重复做题用卷子就够了；软件存在的意义＝做**纸笔做不到**的事。每个学习软件（尤其复习类）必须体现以下三条，至少占主体，不达标即返工：

1. **做纸笔替代不了的事**：可拖动/可点击的交互、即时反馈、状态会变化的模型（如拖动改变图形看面积↔周长一起变、滑动时间轴看时态变化）。
2. **教"思维逻辑"，不是教答案**：用**分步引导/动画**把"为什么这样做"一步步演示出来（如转述句一步步变身并解释每步原因），让孩子学会**怎么想**，而不是记住结论。
3. **会动的图帮助记忆与兴趣**：**可交互的关系脑图、立体/结构示意图、引导图、动画**（如名家名作关系脑图、几何拆补动画、行程动画、记忆翻卡），降低记忆负担、提起兴趣。

> 实现仍是零依赖纯静态（原生 JS + SVG + CSS 动画即可）。题目/测验可以有，但只能是"理解之后的检验"，不能是软件的主体。
> 标杆参考：`apps/kenton_chinese_lab.html`（转述句变身机·分步动画讲逻辑 + 名家名作关系脑图 + 古诗记忆翻卡）。

## 学习记忆层（内容迭代铁律 ⭐）

学习内容（卷子/练习等）需要**长期记忆、逐次迭代**：后面的内容要基于以前的来出。
记忆载体是仓库文件（`data/learning/` + `docs/学习记忆.md`），因为 AI 无跨会话记忆——
每次会话都是重新克隆仓库，**只有提交进 git 的文件才读得到**。因此：

**出任何学习卷子/练习内容前，必须先读：**
1. `docs/学习记忆_哥哥.md` 或 `docs/学习记忆_弟弟.md` —— 🌟该孩子的**长期学习总档**（人物画像、三科长期弱点、错因标签、复习优先级、软件规格、总提示词），最权威
2. `data/learning/textbook_reference.json` —— ⭐**教材对齐总纲**（见下「教材对齐」），定方向与单元结构
3. `data/learning/profile_<who>.json` —— 该孩子的学情画像（各知识点掌握度、薄弱项）
4. `data/learning/papers_log.json` —— 已出过的卷子台账（避免重复、保证循序渐进、难度递进）
5. `docs/学习记忆.md` —— 成长记录概览

### 教材对齐（⭐ 英语为纲，语文/数学同系列）

- **教材**：上海《尖子生笔记·同步教材》（钟书金牌 / 天津人民出版社，沪教版方向，"紧跟新考向"）。
- **年级对应**：哥哥 Kenton = 英语**四年级下**；弟弟 Eddey = 英语**三年级下**。（升学后由家长告知再更新。）
- **结构**：每册 4 个 Module × 3 个 Unit；每 Unit 五栏目：**思维导图(理知识)·单元词汇(巧预习)·解析教材(划重点)·单元复习(抓考点)·拓展阅读(提素养)**；书末 INDEX（核心词汇 + 重点句型/语法）。
- **对齐方式**：按 Module→Unit 组织；词表取自『单元词汇』+INDEX 核心词汇；讲解对齐『解析教材』；
  复习/出题对齐『单元复习』考点与 INDEX『重点句型/语法』清单；阅读对齐『拓展阅读』；
  **一课一练**按"解析教材划重点 → 单元复习抓考点"的节奏，一个 Unit 一练。

**出完之后，必须更新（同一次提交里一起改）：**
1. `papers_log.json` 追加本卷一条（id/日期/学科/知识点/难度/结果）
2. `profile_<who>.json` 按本次表现调整相关知识点 `mastery` 与 `strengths/weaknesses`
3. `docs/学习记忆.md` 顶部追加一段成长日志

> 这条闭环每次都走 = 内容持续进化。漏更新记忆 = 记忆断档，下次就"失忆"。
> 注：本仓库内 AI 不能"后台常驻自动进化"，迭代发生在每次被调用时；
> 若要定时自动跑（如每日出题），属蓝图 P2 的 GitHub Actions 定时任务，另议。

## 常见任务做法（"更新门户"）

- **加今日任务** → 改 `data/tasks.json`（更新 `date` 与对应孩子数组）。
- **入库新软件** → 把 HTML 放进 `apps/`，在 `data/apps.json` 追加一条索引。
- **更新积分** → 改 `data/scores.json`（同时更新 `updated`）。
- **发布播报** → 在 `podcast/` 新建 `YYYY-MM-DD.html`，在 `data/podcast.json` 追加一条。
- **加卷子** → PDF 放进 `papers/`，在对应 `tasks.json` 用 `type:"pdf"` 链接到它，
  **并按上节"学习记忆层"铁律先读后更新** `data/learning/`。

新建软件/播报页时，复用现有页面的窄屏单列结构与 `assets/styles.css`，保持视觉一致。
