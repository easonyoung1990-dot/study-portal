# 每日早新闻 · 自动化说明

把"每天早上 7 点前自动生成财经/科技早新闻并发布到网站"这件事，交给 GitHub Actions 定时跑。
一次配好，之后无人值守。

## 它是怎么跑的

```
GitHub Actions 定时器（每天约北京时间 04:30）
   → 唤醒 Claude（claude-code-action）
   → 联网搜当天财经/科技热点（WebSearch）
   → 按 .github/prompts/morning-news.md 生成 podcast/<日期>.html
   → 追加 data/podcast.json
   → git 提交并 push 到 main
   → GitHub Pages 约 1 分钟后自动上线
```

孩子端的 `today.html` 会自动取 `podcast.json` 的最后一条作"今日早新闻"，无需手动改页面。

## 相关文件

| 文件 | 作用 |
|------|------|
| `.github/workflows/daily-morning-news.yml` | 定时器 + 运行步骤（cron `30 20 * * *` = 北京 04:30，另支持手动触发） |
| `.github/prompts/morning-news.md` | 给"自动跑的 Claude"看的固定指令（取材口味、生成格式、提交方式） |
| `podcast/<日期>.html` | 每天自动生成的当期早新闻页 |
| `data/podcast.json` | 早新闻索引，今日入口自动取最新一条 |

## 你需要做的一步（只做一次）⭐

工作流要调用 Claude，需要一把凭据。这里用**你的 Claude 订阅**生成的 OAuth token
（不按 API 单独计费）。**它存在 GitHub 加密 Secret 里，不会进入仓库代码，符合隐私红线**。

1. 在你电脑的终端运行 `claude setup-token`，按提示用订阅账号登录，复制输出的 token。
2. 打开仓库 **Settings → Secrets and variables → Actions → New repository secret**
3. Name 填 `CLAUDE_CODE_OAUTH_TOKEN`，Value 粘贴上一步的 token。
4. 保存即可。

> 备选：若以后想改用按量付费的 API Key，把 Secret 换成 `ANTHROPIC_API_KEY`，
> 并把 yml 里 `claude_code_oauth_token` 那行改回 `anthropic_api_key` 即可。

> 注：本仓库当前在特性分支 `claude/daily-morning-news-ubexc9` 上开发。定时任务要真正每天跑，
> 需要把本分支合并进 `main`（工作流文件必须在默认分支上，schedule 才会生效）。

## 验证 / 手动跑一期

合并到 `main` 并配好密钥后：

- 进仓库 **Actions → 每日早新闻 → Run workflow** 可手动触发一次，验证是否正常生成并发布。
- 看 `podcast/` 里是否多了当天的 HTML、网站今日早新闻入口是否更新。

## 想调整

- **改时间**：编辑 yml 里的 cron（注意是 UTC，北京时间要减 8 小时）。
- **改口味 / 栏目 / 风格**：编辑 `.github/prompts/morning-news.md` 即可，无需动代码。
- **换模型**：改 yml 里 `--model`（如换成更强的 `claude-opus-4-8`，成本更高）。

## 成本与稳定性提醒

- 每天 1 次调用，用量很小；模型默认用 `claude-sonnet-4-6`，性价比高。
- GitHub 免费版 cron 在高峰期可能延迟十几分钟，已把触发设在 04:30 留足提前量，确保 7 点前上线。
- 偶发失败（网络/限流）不影响第二天；也可随时手动 Run 一次补发。
