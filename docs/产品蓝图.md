# 家庭学习门户 · 产品蓝图 v1.0

> 定位一句话：把散落在对话框、下载文件夹、OPPO 手机里的所有学习资产，
> 收拢成一个全家任何设备打开网址就能用的"自家学习App"。
> 本文档放入仓库 docs/ 目录，是 Claude Code 的总施工图。

---

## 一、用户与核心场景

| 用户 | 设备 | 打开门户要完成的事 |
|---|---|---|
| Kenton（4B） | 平板/手机 | 看今日任务 → 用学习软件 → 听播报 → 看自己星星 |
| Eddey（3B） | 平板/手机 | 同上（内容独立，绝不混用） |
| 爸爸 | 电脑/OPPO | 发布新材料、打印卷子、看错题趋势报告 |

设计基线：**窄屏单列优先**（OPPO 半屏 / 平板竖屏），大按钮，孩子零打字。

## 二、总架构（三层）

```
┌─ 展示层：GitHub Pages 静态网站（免费，自动 HTTPS）
│   孩子和全家访问：https://<用户名>.github.io/study-portal/
├─ 数据层：仓库内 /data/*.json（任务、软件清单、积分、播报索引）
│   敏感明细（错题原文、批改记录）只存本地 local-data/，永不上传
└─ 生产层：你电脑上的 Claude Code（学习工厂）
    生成/更新内容 → git push → 网站 1 分钟内自动更新
```

关键认知：静态网站**只读**。所有数据更新都发生在爸爸电脑端
（一句"更新门户"→ Claude Code 改 JSON → 推送），孩子端永远只是打开看。

## 三、隐私红线（负责人强制条款）

1. 免费版 GitHub Pages 要求仓库公开 → **公开仓库 = 全网可见**，因此：
   - 网站只用英文小名 Kenton/Eddey，绝不出现姓氏、学校、班级、照片
   - 错题原文、批改细节、成绩单**只留本地 local-data/**（.gitignore 屏蔽）
   - 网站上的"错题趋势"只放匿名化统计（如"小数除法正确率 62%→85%"）
2. 家长页不放任何账号密码；仓库里永远不提交任何密钥
3. 若未来想全部私密：迁移到支持私有仓库的免费托管（届时再议，架构不变）

## 四、页面结构（v1 共 5 页）

```
study-portal/
├─ index.html        ① 首页：选择 Kenton / Eddey 两个大按钮
├─ today.html        ② 今日页：日期 + 今日任务清单 + 今日播报入口
├─ apps.html         ③ 软件库：英语阅读软件 / 古诗App / 数学App 卡片墙
│                       （每个软件就是仓库里的一个 HTML，点卡片直接打开）
├─ stars.html        ④ 星星榜：两兄弟本周/累计积分（爸爸端每晚汇总更新）
└─ parent.html       ⑤ 家长页：卷子 PDF 下载打印 + 匿名化错题趋势图
```

## 五、数据设计（/data/ 目录）

```jsonc
// tasks.json —— 今日任务（爸爸端每晚生成次日版）
{ "date": "2026-06-12",
  "kenton": [ {"type":"app","title":"英语阅读·U3","link":"apps/eng_u3.html","stars":3},
              {"type":"pdf","title":"数学小卷·小数除法","link":"papers/k_math_0612.pdf","stars":5} ],
  "eddey":  [ ... ] }

// apps.json —— 软件库索引（新软件入库时追加一条）
[ {"child":"kenton","subject":"英语","title":"英语探险乐园 U3",
   "file":"apps/eng_u3.html","added":"2026-06-11"} ]

// scores.json —— 积分（唯一可写来源：爸爸端汇总）
{ "updated":"2026-06-11", "kenton":{"week":18,"total":236}, "eddey":{"week":21,"total":214} }

// podcast.json —— 播报索引（每期一条，指向当期 HTML）
[ {"date":"2026-06-11","file":"podcast/2026-06-11.html","topics":["NBA","太空"]} ]
```

本地专属（不上传）：`local-data/mistakes.csv` 错题库
（字段：日期/孩子/学科/知识点坐标/错因三级/题目原文/重做结果），
是周报、趋势图、冲刺卷调题的数据源。

## 七、P0 开工指令（直接整段粘给 Claude Code）

```
读取 docs/产品蓝图.md 后执行 P0：
1) 在学习工厂下建 study-portal/ 目录，按蓝图第四、五节建立页面骨架与 data/ 示例
2) index.html：Kenton/Eddey 两个大按钮，糖果色卡通风，窄屏单列
3) apps.html：读取 data/apps.json 渲染软件卡片墙；先放入一个占位示例软件
4) 配置 .gitignore 屏蔽 local-data/
5) 创建公开仓库 study-portal，推送，并开启 GitHub Pages
6) 把最终网址打印给我，并自查蓝图第三节隐私红线是否全部满足
每步开始前用一句中文说明意图。
```

## 八、风险与边界

- **额度**：P2 的定时任务消耗订阅额度，初版频率定为每天 1 次（播报），观察一周再加
- **不做**：实时聊天、账号系统、孩子端写入（静态架构不支持，也不需要）
- **回退**：任何一版改坏，`git` 历史一条命令回退——这就是当初装它的意义
